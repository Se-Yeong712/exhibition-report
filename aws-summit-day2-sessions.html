AI Exhibition Visit Report · 2026

세션 노트로 다시 보는 AI Day,
AWS Summit Seoul 2026 (두 번째 기록)

기조연설 말고 — 둘째 날 브레이크아웃 세션에서 직접 메모한 Bedrock·Kiro·에이전트 이야기.

📅 2026.05.21 (Day 2, AI Day) 📍 코엑스 컨벤션 센터 🎟 Breakout Sessions

앞선 후기가 AI Day의 전체 분위기와 기조연설을 담았다면, 이번엔 시선을 좁혀본다. 둘째 날은 9개의 AI 중심 트랙으로 빼곡했는데, 그중 개발자 입장에서 가장 메모를 많이 남긴 세 세션 — 에이전트 구축, Bedrock 운영, 그리고 AI 시대의 개발 도구 — 을 골라 정리했다.

01 AI Day는 트랙이 9개였다

둘째 날 AI Day는 단일 흐름이 아니라 AI 모델 개발, 관리형 AI 모델 활용, 데이터 기반 AI, 에이전트 구축 도구, 에이전트 실행·전환, AI 시대 진화하는 개발자, AI 인프라, AI 보안·거버넌스, AWS 커뮤니티까지 9개 트랙으로 갈라져 있었다. 세션 레벨도 입문(100)부터 전문가(400)까지 나뉘어 있어, 같은 시간대에도 어느 방에 들어가느냐에 따라 완전히 다른 깊이의 이야기를 들을 수 있었다. 모든 걸 들을 수는 없으니, 나는 '에이전트를 실제로 어떻게 만들고 운영하나'라는 축으로 동선을 짰다.

02 가장 메모가 많았던 세 세션

Session · Agent Build

에이전트는 '데모'가 아니라 '운영'의 문제였다

에이전트 구축 트랙에서 반복해서 강조된 건, 프로토타입을 만드는 건 쉬워졌지만 그걸 안정적으로 굴리는 건 전혀 다른 문제라는 점이었다. 도구 호출(tool use)이 실패했을 때의 재시도, 세션 메모리 관리, 권한 통제 같은 '운영의 디테일'이 결국 에이전트의 완성도를 가른다는 이야기가 인상 깊었다.

Session · Managed Models

Bedrock — 보안이 제품에 녹아 있다

관리형 모델 트랙에서는 Amazon Bedrock이 중심이었다. 특히 인터넷을 거치지 않고 기업 VPC 내부에서 모델 API를 호출하는 구조가 눈에 들어왔다. 금융·의료처럼 네트워크 보안 규정이 엄격한 곳에서는 '모델 성능'보다 '데이터가 어디로 나가느냐'가 먼저인데, 그 고민이 제품 설계에 이미 반영돼 있었다. 엑스포의 한 생의학 스타트업 부스에서도 Bedrock 기반 추론·에이전트 도구 호출·세션 메모리를 묶은 아키텍처를 실제로 볼 수 있었다.

Session · Evolving Developer

Kiro — 자동완성을 넘어 '스펙'을 다루는 IDE

'AI 시대 진화하는 개발자' 트랙에서 가장 흥미로웠던 건 Kiro였다. 기존 AI 코딩 도구가 "이 함수 완성해줘" 수준의 자동완성에 머물렀다면, Kiro는 개발자가 자연어로 요구사항(Spec)을 쓰면 그걸 분석해 태스크로 분해하고, 코드 작성부터 테스트·문서화까지 일련의 작업을 스스로 수행하는 방향을 지향했다. 세션에서는 MCP, Steering, Powers를 중심으로 효율적인 사용법을 소개했다.

03 반복해서 나온 단어 — MCP

세 세션을 가로지르며 계속 등장한 단어가 MCP(Model Context Protocol)였다. Anthropic이 정의한 일종의 표준 프로토콜로, AI 에이전트 같은 MCP 호스트가 클라이언트를 통해 하나의 표준 인터페이스로 외부 서비스·도구에 연결되도록 한다는 개념이다. 도구마다 제각각이던 연결 방식을 표준 하나로 묶는다는 발상인데, 에이전트가 '말만 하는 AI'에서 '실제로 일을 처리하는 AI'로 넘어가는 데 이 표준화가 꽤 중요한 역할을 하겠다는 생각이 들었다.

결국 AI Day의 세션들은 한 방향을 가리키고 있었다 — 모델을 더 똑똑하게 만드는 경쟁에서, 그 모델을 안전하고 표준화된 방식으로 '일하게' 만드는 경쟁으로.

04 다녀와서 — 정리하며

기조연설이 "AI가 동료가 되는 시대"라는 큰 그림을 줬다면, 브레이크아웃 세션은 그 그림을 떠받치는 부품들을 보여줬다. 에이전트의 운영 디테일, Bedrock의 보안 설계, Kiro의 스펙 기반 개발, 그리고 이 모두를 잇는 MCP. 추상적인 비전보다 이런 구체적인 조각들이 오히려 더 오래 기억에 남았다.

다음에 같은 행사를 간다면, 이번에 시간이 겹쳐 못 들어간 'AI 보안·거버넌스' 트랙을 챙겨 듣고 싶다. 데이터가 새지 않게 하는 문제는 결국 모든 산업이 공통으로 부딪히는 벽이니까.
